# Документация по гиту
git status - список файлов и их статус
git commit -m "текст коммита"
Флаг -m означает message. Без него нельзя сделать коммит.
git checkout - переключает на что-то, например ветку или коммит
Пример: git checkout -b name/name 
переключает на ветку и фрагом -b создаёт её.

Важное правило, в main ветке не должно быть мусора. Для новвых фич создаём новые ветки.


# Оформление коммитов
feat: новая функция или фича
Пример: feat: добавлен ввод имени игрока

fix: исправление бага
Пример: fix: исправлена утечка памяти

docs: изменения в доках
Пример: добавлен новый раздел в документацию

chore: рутинные задачи
Например: создание проекта cargo или что-то такое
